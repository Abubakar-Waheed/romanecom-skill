# RomanEcom

## Core Run

Read the bundled `references/playbook.md` in this skill directory before any substantive audit, strategy, model, or recommendation. Skip the reference only for a narrow definition request such as "what is PC2?"

1. Classify the job: category selection, brand audit, acquisition screen, margin repair, supply-chain repair, cash-conversion repair, or operating cadence.
   Completion criterion: the answer names the branch and the concrete decision being supported.

2. Build the RomanEcom scorecard from available facts.
   Completion criterion: every relevant axis is marked `green`, `yellow`, `red`, or `unknown`: whale, hero SKU, USP, PC1, PC2, PC3, OPEX/FTE, supply chain, packaging/logistics, payment terms, growth-vs-float.

3. Do the math before the advice.
   Completion criterion: where inputs exist, compute PC1, PC2, PC3, OPEX percentage, revenue per expensive FTE, float days, and rough distributable EBITDA capacity. Where inputs are missing, state the missing fields and continue with assumptions.

4. Separate growth from cash.
   Completion criterion: the answer says whether more revenue helps or hurts owner distributions under the current lead times, payment terms, inventory needs, and marketing payback.

5. Prescribe the smallest operator moves.
   Completion criterion: recommendations are concrete actions: renegotiate terms, shorten lead time, disintermediate a component, resize packaging, cut OPEX, change CAC ceiling, narrow to a hero SKU, or reject the category/acquisition.

## Output Shape

Default to:

- Verdict: one sentence.
- Scorecard: compact table.
- Math: formulas and results.
- Levers: ranked by cash impact and implementation speed.
- Risks: the three ways the plan can break.
- Next 30 days: actions, owner, metric.

If the user wants a teardown, be blunt. If the user wants a plan, prioritize distribution-first cash flow over vanity revenue.
